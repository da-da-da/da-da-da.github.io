XML-可扩展标记语言

数据表示格式，描述非常复杂的数据格式，常用于传输和存储数据，文件为.xml

特点： 1. 纯文本，默认UTF-8 2. 可嵌套，适合结构化数据

固定格式

  • 首行<?xml version="1.0"?>

  • 如果以类似<!DOCTYPE note SYSTEM "book.dtd">声明的是文档定义类型（DTD：Document Type Definition），DTD是可选的。

    DTD（文档类型定义）的作用是定义 XML 文档的合法构建模块。

    DTD 可被成行地声明于 XML 文档中，也可作为一个外部引用。

  • 一个XML文档有且仅有一个根元素，根元素可以包含任意个子元素，元素可以包含属性，例如，<isbn lang="CN">1234567</isbn>包含一个属性lang="CN"，且元素必须正确嵌套。如果是空元素，可以用<tag/>表示。

由于使用了<、>以及引号等标识符，如果内容出现了特殊符号，需要使用&???;表示转义。例如，Java<tm>必须写成：

格式正确的XML（Well Formed）是指XML的格式是正确的，可以被解析器正常读取。而合法的XML是指，不但XML格式正确，而且它的数据结构可以被DTD或者XSD验证。

为什么使用 DTD？

通过 DTD，您的每一个 XML 文件均可携带一个有关其自身格式的描述。

通过 DTD，独立的团体可一致地使用某个标准的 DTD 来交换数据。

而您的应用程序也可使用某个标准的 DTD 来验证从外部接收到的数据。

您还可以使用 DTD 来验证您自身的数据。

java正则表达式

发表于 2020-08-09 | 分类于 java易混淆问题

在其他语言中，\\ 表示：我想要在正则表达式中插入一个普通的（字面上的）反斜杠，请不要给它任何特殊的意义。

在 Java 中，\\表示：我要插入一个正则表达式的反斜线，所以其后的字符具有特殊的意义。

所以，在其他的语言中（如Perl），一个反斜杠 \ 就足以具有转义的作用，而在 Java 中正则表达式中则需要有两个反斜杠才能被解析为其他语言中的转义作用。也可以简单的理解在 Java 的正则表达式中，两个 \\ 代表其他语言中的一个 \，这也就是为什么表示一位数字的正则表达式是 \\d，而表示一个普通的反斜杠是 \\\\。

如果输入\\\的话，会报错，\必须要和一个字符组合在一起表示该字符的含义那前两个\\是表示\，第三个\没有与之匹配的东西，所以四个\组合在一起，表示了两个转义\的字符串

在Java正则表达式中，要表示\，正则表达式是\\，在Java中需要 \\\\来转义后，变成匹配\的正则表达式\\

LeetCode-100

发表于 2020-07-28 | 分类于 二叉树

给定两个二叉树，编写一个函数来检验它们是否相同。

如果两个树在结构上相同，并且节点具有相同的值，则认为它们是相同的。

示例 1:

输入: 1 1
/ \ /
2 3 2 3

[1,2,3],   [1,2,3]

输出: true
示例 2:

输入: 1 1
/
2 2

[1,2],     [1,null,2]

输出: false

来源：力扣（LeetCode）
链接：https://leetcode-cn.com/problems/same-tree

  • 深度优先算法

    思路：对于两个节点，以他们为根节点的两个树，要判断这两个节点的左子树是否相等，右子树是否相等，对于每个节点都是如此判断，这是一个递归的过程，则使用深度优先算法，递归判断两个节点是否相等，其左右子树是否相等。

    复杂度：时间复杂度：O(min(m,n))，其中 m 和 n 分别是两个二叉树的节点数。对两个二叉树同时进行深度优先搜索，只有当两个二叉树中的对应节点都不为空时才会访问到该节点，因此被访问到的节点数不会超过较小的二叉树的节点数。

    空间复杂度：O(min(m,n))，其中 m 和 n 分别是两个二叉树的节点数。空间复杂度取决于递归调用的层数，递归调用的层数不会超过较小的二叉树的最大高度，最坏情况下，二叉树的高度等于节点数。

    判断什么时候要结束递归

    • 两个节点都为null，return true
    • 两个节点之一为null，return false
    • 节点不相等，return false

  • 广度优先算法

    这个方法是看了题解才知道可以用广度

    思路：从根节点开始进行层序遍历，先看出队列的值是否相同，再看每个出队列节点的的左右子节点的结构，如果不同则false，如果相同入栈。直到一个队列为空

    复杂度分析

    时间复杂度：O(min(m,n))，其中 m 和 n 分别是两个二叉树的节点数。对两个二叉树同时进行广度优先搜索，最坏是遍历完节点数较小的那个

    空间复杂度：O(min(m,n))，其中 m 和 n 分别是两个二叉树的节点数。空间复杂度取决于队列中的元素个数，队列中的元素个数不会超过较小的二叉树的节点数。
